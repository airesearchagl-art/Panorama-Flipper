# Panorama Flipper

ブラウザだけで動く、360°エクイレクタングラー（equirectangular）パノラマ画像の変換ツールです。左右反転・180°回転・90°回転に対応し、複数画像の一括変換・一括ダウンロードが可能です。

## 解決する課題

D5 Render、Twinmotion、Lumion、Enscape、SketchUp系のレンダリングパイプラインから書き出した360°パノラマ画像が、左右反転した状態で出力されることがあります。ビューア側の設定で都度補正するのは手間がかかり、配布先によって表示が揃わない原因にもなります。Panorama Flipperは変換処理を画像ファイル自体に焼き込むため、どのビューアで開いても一貫した見え方になります。

## 主な機能（v1.2）

- **複数画像の一括読み込み**：ファイル選択またはドラッグ＆ドロップで複数画像を同時に読み込み
- **変換モードの選択**：左右反転 / 180°回転 / 90°回転 / -90°回転
- **画像リスト表示**：読み込んだ画像をサムネイル付きで一覧表示、変換状態をバッジで確認
- **選択画像の変換と一括変換**：1枚ずつ、またはすべてまとめて変換
- **元画像 / 変換後のプレビュー比較**
- **PNG / JPGダウンロード**：選択画像を個別にダウンロード（JPEGは品質0.92）
- **ZIP一括ダウンロード**：変換済みの全画像をPNG/JPGのZIPでまとめてダウンロード（JSZip使用）
- **画像情報パネル**：ファイル名、画像サイズ、アスペクト比、パノラマ判定、ファイル形式
- **リセット**：初期状態に戻す
- 画像処理は100%ブラウザ内で完結し、サーバーへのアップロードは行わない

## 変換モードの詳細

| モード | 出力サイズ | 処理 |
|--------|-----------|------|
| 左右反転 | 元と同じ | 水平ミラー反転 |
| 180°回転 | 元と同じ | 180°回転 |
| 90°回転 | 幅↔高さを入れ替え | 時計回り90°回転 |
| -90°回転 | 幅↔高さを入れ替え | 反時計回り90°回転 |

## ファイル名の命名規則

| 変換 | 例（PNG） | 例（JPG） |
|------|----------|----------|
| 左右反転 | sample_flipped.png | sample_flipped.jpg |
| 180°回転 | sample_rot180.png | sample_rot180.jpg |
| 90°回転 | sample_rot90.png | sample_rot90.jpg |
| -90°回転 | sample_rot-90.png | sample_rot-90.jpg |

ZIPファイル名例：`panorama-flipper_flipped_png.zip`

## 使い方

1. アプリを開く（ローカル、またはVercel公開URL）。
2. 画像をドラッグ＆ドロップ、または「ファイルを選択」をクリック（複数可）。
3. 変換モードを「左右反転」「180°回転」「90°回転」「-90°回転」から選ぶ。
4. 一覧から変換したい画像をクリックして選択。
5. 「選択画像を変換」または「すべて変換」をクリック。
6. 個別ダウンロード、またはZIP一括ダウンロードで保存。
7. 「リセット」で初期状態に戻す。

## 90°回転時の注意

90°回転および-90°回転は画像の幅と高さを入れ替えるため、2:1比率の360°パノラマ画像ではほとんどのパノラマビューアで正しく表示されません。通常のパノラマ補正には「左右反転」または「180°回転」を使用してください。

## ローカルでの起動方法

ビルドや依存パッケージは不要です。

```bash
git clone https://github.com/airesearchagl-art/Panorama-Flipper.git
cd Panorama-Flipper
```

`index.html` をブラウザで直接開くか、任意の静的サーバーで配信してください。

```bash
python3 -m http.server 8000
```

`http://localhost:8000` でアクセスできます。

## Vercelへのデプロイ方法

1. Vercelダッシュボードを開く。
2. 「Add New Project」をクリック。
3. このGitHubリポジトリ（`airesearchagl-art/Panorama-Flipper`）をImportする。
4. Framework Presetを「Other」に設定。
5. Build Commandは空欄のままにする。
6. Output Directoryは `.` または空欄に設定。
7. Production Branchを `main` に設定。
8. 「Deploy」をクリック。

このプロジェクトはビルドステップ不要の静的アプリです。デプロイ後は `https://xxxx.vercel.app` のようなURLでアクセスでき、以後 `main` へpushすると自動デプロイされます。

## 使用ライブラリ

| ライブラリ | 用途 | 読み込み方法 |
|-----------|------|-------------|
| [JSZip](https://stuk.github.io/jszip/) 3.10.1 | ZIP一括ダウンロード | CDN（jsDelivr） |

JSZipが読み込めない場合は、個別ダウンロードにフォールバックします。

## プライバシー方針

すべての画像処理はブラウザのCanvas APIを使ってローカルで実行されます。読み込んだ画像はサーバーや第三者サービスに送信されることはありません。

## ビューアでの見え方に関する注意

Panorama Flipper上で変換後のプレビューを確認できた場合、画像ファイル自体の変換処理は完了しています。別の360°ビューアで意図と異なる表示になる場合は、そのビューア側の球体マッピング処理、左右反転（ミラー）設定、または表示仕様によるものである可能性があります。お使いのビューアの表示設定をご確認ください。
